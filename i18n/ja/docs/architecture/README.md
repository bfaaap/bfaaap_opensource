# アーキテクチャ

> 🌐 [English](../../../../docs/architecture/README.md) · **日本語** · [Deutsch](../../../de/docs/architecture/README.md)

## 信号の流れ

1. **頭部トラッキング（iOS）。** コントローラアプリが ARKit `ARFaceTrackingConfiguration`
   （TrueDepthカメラ）で、ユーザーの頭のピッチ角を読み取る。
2. **マッピング。** 角度にユーザーの「倍率」（スライダ 0〜20）を掛け、`0〜99` にクランプ。
3. **送信（BLE）。** 値を ASCII 文字列 `i00`〜`i99` として **Nordic UART Service (NUS)** で送信。
   踏み込み/戻しの遷移ではマーカーバイト `N`（オン）・`F`（オフ）も送る。
4. **駆動（装置）。** Pico ファームウェアが値を解釈し、**IQ / Fortiq** モーターに比例してサステインペダルを
   押すよう指令する。

## BLE 詳細（実測）

| 項目 | 値 |
|------|-----|
| サービス (NUS) | `6E400001-B5A3-F393-E0A9-E50E24DCCA9E` |
| Write (RX, アプリ→装置) | `6E400002-B5A3-F393-E0A9-E50E24DCCA9E` |
| Notify (TX, 装置→アプリ) | `6E400003-B5A3-F393-E0A9-E50E24DCCA9E` |
| その他 | DFU `00001530-…`、Device Information `180A`、Battery `180F` |
| アドバタイズ名 | `bFaaaPSwitch_1`（`_1`〜`_4` に改名可＝複数台/チャンネル用） |

アプリはアドバタイズのローカル名でフィルタし、`bFaaaPSwitch_n` 装置にのみ接続して、近くの他の
ペリフェラルは無視します。

## ペイロード形式

- `iNN` —— 目標値、`NN` = `00`〜`99`（角度 × 倍率、クランプ済み）。
- `N` —— 踏み込み信号（ペダル作動）。
- `F` —— 戻し信号（ペダルがニュートラルへ）。
- 値はタイマーで連続送信（現行ファーム/アプリで約100ms間隔）。
