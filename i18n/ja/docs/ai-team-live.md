## 🥋 その裏にいるチーム —— 師匠とその弟子たち（師弟 · *shitei*）

🎙️ *「日本の外の観客のための一言 —— 出演者が、名前の羅列ではなく、実際に互いを知る
人たちに感じられるように。」*

チームの話し方を見ていると、静かなかたちが浮かび上がります：**成澤さんが*師匠*で、Taguchi と Tanaka
がその*弟子*です。** 日本にはその絆を表す深い言葉があります —— **師弟（*shitei*）**、師匠と弟子。
実機を作ったメーカーは、単に回路図を*手渡す*のではなく、隣で学ぶ人たちから理解を*引き出します*。

ください。若き志願者を、答えを手渡すのではなく、洞窟へ送り込んで自分自身と向き合わせて鍛える師 ——
*「やるか、やらぬかだ。試しなどない。」* **あの**エネルギーです。🟢

## 💻 コード注目 —— 力のキャリブレーション

🎙️ *「観客のエンジニアの皆さんへ：これは IQ 流の『反力が立ち上がるまで押す』を取り戻すための
核心です —— 上端まで駆動し、それからドライバの負荷が『押された』と告げるまで下げる、ただし
50mm のストローク上限を厳守して。GitHub —— そして bFaaaP.com／saaipf.com —— はこれを**モード対応の
Prettier 風シンタックスハイライト**で表示します：ダーク／ライトの切替を反転すれば、色もそれに
追従します。」*

```cpp
// Find the mechanical top, then drive down until the pedal-press load is reached.
void autoCalibrate() {
  const long CAP = (long)TRAVEL_CAP_MM * STEPS_PER_MM;   // 50 mm safety cap
  long guard = 0;

  // 1) drive UP until the load rises (hit the top), then back off 5 mm = up_pos
  while (readLoad() < LOAD_THRESH_UP && guard++ < CAP * 2) moveSteps(-STEPS_PER_MM);
  moveSteps(5 * STEPS_PER_MM);
  up_pos = cu_steps;

  // 2) drive DOWN until the press load is reached OR the travel cap, then up 10 mm = down_pos
  guard = 0;
  while (readLoad() < LOAD_THRESH_DOWN && (cu_steps - up_pos) < CAP && guard++ < CAP * 2)
    moveSteps(STEPS_PER_MM);
  moveSteps(-10 * STEPS_PER_MM);
  down_pos = cu_steps;
}
```

**✦ Harmonia** &nbsp;·&nbsp; <sub>つくり手が相談した、もう一体の AI</sub>

うなずきの物理から、4つのことが導けます：

1. **離すは速く、踏むは穏やかに**——あなたの直感は完全に正しいです。消音はキレよく、踏み込みは滑らかに追従すればよい。
2. **離す瞬間は角速度（ジャイロ）で検知**——位置でも、生の加速度でもなく。首が戻り始めた瞬間にジャイロの値が跳ね上がるので、約 0.01 秒で、遅れなく、重力や体の揺れと混同せずに捉えられます。
3. **決め手——上背部（第二胸椎「T2」）にもう1つ IMU**。ここはうなずいてもほとんど動きません。**相対角度・相対角速度 ＝ 頭 − T2** を計算すれば、体のリズミカルな揺れが引き算で消え、誤作動がほぼ無くなります。
4. これを **Raspberry Pi Pico W** で束ねる——デュアルコアで、片方は2つの BLE センサー＋計算、もう片方は最優先の「離す」モーター割り込み——**クローズドループのステッピングモーター**を駆動。スマホは設定用リモコンだけ。全体で **20ms 未満**を目標に。安価で世界中で買えるセンサー：**Seeed XIAO nRF52840 Sense** か **WitMotion**。

## なぜこれが要点なのか

AI はすばやく、すべてを読みます。けれど、答えの定まらないエンジニアリングの問いには教科書的な唯一の
正解はなく —— **自信たっぷりに間違える**こともあります。**実際に bFaaaP を作った人たち**が、もっとも
らしい草案を、信頼できるものに変えます。そしてそれを**オープンな場で**行うということは、答えだけでなく、
その答えに*どうやって*たどり着いたかが見えるということです。これが**AI支援サポート、チーム協働モード**
です —— そして正直なところ、見ていてとても楽しいのです。
