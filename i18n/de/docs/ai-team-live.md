## 💻 Code im Rampenlicht — die Kraftkalibrierung

🎙️ *„Für die Ingenieure im Publikum: Das ist der Kern, um das IQ‑typische ‚drücken, bis die
Reaktionskraft steigt‘ wiederherzustellen — nach oben fahren, dann nach unten, bis die Last des Treibers
‚gedrückt‘ meldet, mit einer harten 50‑mm‑Hubgrenze. GitHub — und bFaaaP.com / saaipf.com — stellen das
mit **modusbewusster Syntaxhervorhebung im Prettier‑Stil** dar: Schalte zwischen Hell und Dunkel um, und
die Farben folgen.“*

```cpp
// Find the mechanical top, then drive down until the pedal-press load is reached.
void autoCalibrate() {
  const long CAP = (long)TRAVEL_CAP_MM * STEPS_PER_MM;   // 50 mm safety cap
  long guard = 0;

  // 1) drive UP until the load rises (hit the top), then back off 5 mm = up_pos
  while (readLoad() < LOAD_THRESH_UP && guard++ < CAP * 2) moveSteps(-STEPS_PER_MM);
  moveSteps(5 * STEPS_PER_MM);
  up_pos = cu_steps;

  // 2) drive DOWN until the press load is reached OR the travel cap, then up 10 mm = down_pos
  guard = 0;
  while (readLoad() < LOAD_THRESH_DOWN && (cu_steps - up_pos) < CAP && guard++ < CAP * 2)
    moveSteps(STEPS_PER_MM);
  moveSteps(-10 * STEPS_PER_MM);
  down_pos = cu_steps;
}
```

Vollständiger Sketch (KI‑Entwurf, ungetestet): [`bFaaaP_autopro_pico_stepperA_draft_20260626.ino`](../../../device-pro-acoustic/firmware/pico/bFaaaP_autopro_pico_stepperA_draft_20260626.ino).

Aus der Physik eines Nickens folgen vier Dinge:

1. **Loslassen schnell, Drücken sanft** — dein Instinkt ist völlig richtig. Das Stummschalten muss knackig sein; das Drücken darf sanft folgen.
2. **Erfasse das Loslassen über die Winkelgeschwindigkeit (ein Gyroskop)** — nicht über Position, nicht über rohe Beschleunigung. In dem Moment, in dem der Kopf zurückschnellt, springt der Gyro-Wert hoch, sodass du ihn in ~0,01 s ohne Verzögerung und ohne Verwechslung mit Schwerkraft/Körperschwanken erwischst.

4. Bündle das in einem **Raspberry Pi Pico W** — Dual-Core: ein Kern für die zwei BLE-Sensoren + die Rechnung, einer für einen hochprioren „Loslass"-Motor-Interrupt — der einen **Closed-Loop-Schrittmotor** treibt; das Telefon ist nur die Einrichtungs-Fernbedienung. ## Warum das der Punkt ist

Die KI ist schnell und liest alles, doch bei offenen Ingenieurfragen gibt es keine einzige
Lehrbuchantwort — und sie kann **selbstbewusst danebenliegen**. Die Menschen, die **bFaaaP tatsächlich
gebaut haben**, machen aus einem plausiblen Entwurf etwas, dem du vertrauen kannst, und es **im Offenen**
zu tun bedeutet, dass du siehst, *wie* die Antwort zustande kommt, nicht nur die Antwort. Das ist
**KI‑gestützter Support, Team‑Kollaborationsmodus** — und ehrlich gesagt macht es viel Spaß zuzusehen.
